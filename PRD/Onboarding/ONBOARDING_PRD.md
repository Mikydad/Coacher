# SidePal Onboarding Product Requirements Document

**Version:** 1.0\
**Status:** Ready for Implementation\
**Platform:** Flutter (iOS First)

------------------------------------------------------------------------

# Vision

The onboarding experience should feel less like setting up an app and
more like beginning a personal transformation.

Users should go through an emotional journey:

**Hope → Recognition → Understanding → Trust → Personalization →
Commitment**

By the end of onboarding, users should genuinely feel:

> **"This AI understands me, knows why I've been struggling, and can
> finally help me become the person I want to be."**

The onboarding should never feel like a traditional questionnaire. Every
screen should educate, motivate, or demonstrate value.

------------------------------------------------------------------------

# Design Direction

The overall visual experience should feel like a combination of Apple,
Linear, Arc Browser, Notion, and Headspace.

Characteristics:

-   Premium dark theme
-   Large rounded cards
-   Beautiful soft gradients
-   Large minimal illustrations
-   Generous spacing
-   Strong typography
-   Smooth page transitions
-   Modern micro animations
-   Clean, uncluttered layouts
-   Emotionally engaging
-   Calm and futuristic

------------------------------------------------------------------------

# UX Principles

-   Every screen should accomplish one emotional goal.
-   Never overload users with information.
-   Alternate between Recognition → Education → Hope → Solution →
    Personalization → Commitment.
-   Show value before asking for payment.
-   Demonstrate features instead of describing them.
-   Teach users through interaction.
-   Every animation should feel smooth, intentional, and premium.
